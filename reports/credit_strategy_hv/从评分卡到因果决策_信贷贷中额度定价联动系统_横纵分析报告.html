摘要：要研究的不是一个“动态定价模型”，而是一套 constrained decisioning system

本报告的结论可以压缩成一句话：

Current offer 是 decision-time state 的一部分。policy 可以 condition on current offer，但不能把历史相关性当作 action effect。joint action value 必须由可识别的 counterfactual 支撑，并在 hard constraints、state transition 与 uncertainty 下优化。

原对话从“策略如何联动”一路追问到“额度/价格敏感如何识别”“敏感度为什么会变”“current offer 入模会不会内生”“策略实际如何开发”。这已经不是单一 scorecard 或 pricing model 能回答的问题。更准确的研究对象是：Credit Line × Pricing 联动的 decisioning system。它需要同时处理 state、risk、demand、contract action、long-term value、causal identification、policy evaluation 与 consumer protection。

本研究得到六项核心判断：

  1. [推断] Risk scoring 和 action selection 是两个不同的问题。 PD(x) 回答 historical policy 下的风险排序；decisioning system 要比较同一账户在不同 feasible actions 下的增量结果，并选择 long-term net value 最高的 action。
  2. [事实] 额度和价格都会改变行为，而且效应高度异质。 随机与准实验显示，额度会改变借款/消费，价格弹性会随客群和时间窗口变化；但这些研究不能直接给出跨市场通用的调额比例、调价步长或锁定周期。Aydin 额度随机试验、Agarwal 等额度阈值研究、Karlan 与 Zinman 长期价格随机试验
  3. [推断] offer endogeneity 不能靠“删掉 offer 特征”解决。 删除 current offer 会漏掉重要 state；直接使用又可能复制 historical policy。正确分工是把 current/historical offer 与 market context 放进 decision-time state，再用 experiment、quasi-experiment、DML/DR 或 OPE 识别“提额、降价、不动”等 action 的效果。
  4. [推断] Rules、predictive model、CATE、OPE 与 bandit 不在同一抽象层。 Rules 定义 feasible action set，predictive model 估计 conditional outcomes，CATE 比较 action effects，OPE 评估 candidate policy。contextual bandit 只适用于近似 one-step、或长期效应已被可靠封装进 reward 的局部问题；显著的跨期 feedback 应交给 DTR/MDP/POMDP 与 sequential OPE。
  5. [推断] 不存在“直接上 contextual bandit”的通用捷径。 没有稳定的 action semantics、candidate set、decision-time logged propensity、matured long-term outcomes、足够的 overlap/support 和可 replay 的 policy version，online learning 只会更快复制 historical policy 的偏差。
  6. [推断] 公开监管和研究材料没有提供跨法域通用的周期、step size、holdout ratio 或 drift threshold。 这些都属于 context-specific design parameters，应按适用法律、target estimand、sample size、outcome maturity curve、risk appetite、overlap 与 recoverability 校准。美联储 SR 26-2、EBA 贷款发放与监测指引

一、研究问题与理论边界

1.1 研究对象与边界

这里的 Credit Line × Pricing decisioning system，面向存量借款人或账户，在贷中阶段综合 risk、affordability、demand、value、current contract、historical response 与 market context，联合选择额度、价格/综合费率、期限及必要的触达方式，并持续用 matured outcomes 验证 policy。

它不等于以下任何单一对象：

  • 不是“风险分决定额度”的评分卡；
  • 不是“风险越高价格越高”的风险加点表；
  • 不是“预测谁会借”的响应模型；
  • 不是“给每个人一个敏感度标签”的画像系统；
  • 也不是不受约束、自动探索所有价格和额度的强化学习系统。

设 decision history 为 H_t，current offer 为 C_t=(L_t,p_t,q_t)，分别表示额度、价格/综合费率与期限。policy 指从 state/context 到 action 的决策规则；A_t 表示本次 action。A_t 可以编码为目标合同 C_t*，也可以编码为相对变化 ΔC_t=C_t*−C_t，但同一个 estimand 与 policy version 内必须二选一；触达或 execution channel 另记为 M_t。policy 可以 condition on C_t，但不能把 C_t 的历史系数解释为本次合同变更的 causal effect。

系统目标应写成受约束的长期增量价值，而不是单一 PD 或单次响应率：

Action Value(x,a)
= Δ利息与费用收入
- Δ预期信用损失
- Δ资金成本
- Δ运营与补贴成本
- Δ资本占用
+ Δ可归因留存与后续借款价值

并满足 affordability、legal pricing、contract change、risk exposure、funding/capital、fairness、action frequency 与 explainability 等约束。

这里必须防止 double counting：如果 Action Value 已覆盖完整的 discounted multi-period cash flow，continuation value 只能补充尚未展开的未来价值；未来各期 cash flow 已进入 cumulative reward 时，不能再次叠加 retention value。Risk 也有两种角色：expected credit loss 是 reward 中的经济成本，tail loss、concentration 或 maximum exposure 则属于独立约束；二者不能互相替代。

1.2 证据使用规则

本报告区分五类数字：监管硬约束 / 研究 estimate / 内部 observation / 经验 default / 待验证 hypothesis。没有标签的数字不能支撑 decision claim。原对话和研究文件用于确定问题、方案假设与核验清单；法规原文、论文和官方项目文档用于外部证据。任何 estimate 都不能跨产品、跨市场或跨 action dose 直接搬用。

二、纵轴：从“判断人”到 action selection 的八十年演进

[推断] 这段历史不是“scorecard 被新算法淘汰”的路线图，而是 decision object 不断扩大的过程：先量化 risk，再把账户描述为随时间变化的 state，把额度与价格视为会改变行为的 contract action；只有识别出 action effect，系统才有资格讨论 constrained policy learning。

2.0 五条谱系不是接力赛，而是逐步汇流的五个问题

额度—定价联动系统没有单一发明人，也没有一个可以被称为“诞生时刻”的年份。它由五条长期各自发展的谱系汇流而来。每条谱系都解决了前一代系统无法回答的问题，也都带着自己的盲区进入今天。

谱系 它持续追问的问题 留给现代系统的核心资产 单独使用时的盲区
风险测量 谁更可能履约，损失有多大 可验证的风险排序，后续扩展为 PD、LGD、EAD 等量化语言 预测历史政策下的结果，不直接回答该采取什么动作
合同经济学 额度、价格、期限为何同时改变需求、客群和偿付行为 把 offer 视为干预，而不只是风险的被动报价 经济机制成立，不等于个体效应已经被数据识别
行为状态 同一账户今天与数月后为何不是同一个决策对象 动态状态、账户管理、champion/challenger 与反馈闭环 传统分群和规则常只能处理少量离散动作，跨 action interaction 不足
因果识别 观察到的结果中，究竟有多少由动作造成 potential outcomes、异质性 treatment effect、准实验、DML/DR 等识别工具 方法不能凭空补回未观测混杂或历史从未尝试的动作
policy learning 与治理 在不确定、预算、风险、公平和解释约束下如何选动作并持续学习 OPE、保守探索、policy version、审计、监控与回滚 reward、约束或日志设计错误时，学习速度只会放大错误

五条谱系在今天汇入同一条决策事件链。决策前已经生效的合同记为 C_t；风险、余额、历史响应和环境等可观测状态记为 S_t；需求、流动性、偏好等不可完整观测状态记为 Z_t。策略先分配合同动作 R_t，展示与客户接受环节记为 D_t，真正落到合同并生效的动作才是 A_t。因此历史数据不是简单的“状态—结果”，而是：

(C_t, S_t, Z_t) → R_t → D_t → A_t → 多时间窗结果 → (C_{t+1}, S_{t+1}, Z_{t+1})

其中 Z_t 不能被策略直接观测，只能由历史形成信念或区间。R_t、D_t 和 A_t 也不能被一个“offer”字段混写：分配效应、接受机制与 executed action 对应不同 estimand。动作编码必须在一个 policy version 内固定为两种语义之一：目标合同，或相对 C_t 的变化；两种语义只有在决策时 C_t 被完整保存时才能可靠转换。

这五条线并非按年代整齐交棒。治理在评分工业化时已经出现，合同经济学在在线学习之前几十年就指出“价格会改变风险”，champion/challenger 也早于现代 bandit。历史真正呈现的是一组不断加深的追问：能否一致地判断风险；能否及时描述账户；能否理解合同动作；能否识别 action effect；能否在约束内学习更好的政策。

2.1 1941 前后：人工判断开始变成可计算风险

大众消费信贷早期面对的难题不是模型不够复杂，而是人工判断无法复制。信贷员根据职业、收入、居住稳定性和个人印象决定是否授信，少量案件尚可运转；当客户、网点和产品增多，同一人可能在不同审贷员手里得到不同结论，机构也很难知道哪一套经验真的有效。

[事实] David Durand 为 NBER 所做的 1941 年研究《Risk Elements in Consumer Instalment Financing》是统计化消费信贷的重要早期节点。其技术版不仅讨论风险因素，还专门包含信用评级公式、判别函数、显著性与抽样误差等内容。NBER 技术版与目录、风险研究方法章节

主流范式由此发生一次根本变化：历史上的“好贷款/坏贷款”被转化为可比较样本，经验线索被转化为变量和权重，模型的优劣开始接受样本外表现而非专家资历的检验。它解决了规模化授信最基础的两件事——一致性与可验证性。风险从个人判断变成了可以被计算、复核和批量执行的对象。

但成功也锁定了问题的形状。评分公式天然围绕“谁会坏”设计，训练标签来自过去已经发放的贷款，动作通常只有“批或不批”。它既没有描述账户如何随时间变化，也不比较“对同一个人给不同额度或价格会发生什么”。[推断] 风险测量谱系从一开始就是优秀的预测工程，却不是 action-value model；后来的许多系统误把风险分直接映射为额度和价格，正是把这个历史起点当成了终点。

当统计评分证明风险可以被量化，下一个瓶颈便从“能不能算”转向“能不能在大规模机构中稳定使用，并让被影响的人与监管者看得懂”。评分工业化与可审计治理由此进入主线。

2.2 1956—1970s：评分成为工业基础设施，治理也同时进入模型

统计方法要进入大众金融，仅有论文还不够。机构需要稳定的数据流程、可移植的开发方法、批量计算能力和可以长期更新的决策接口。[事实] Fair Isaac 由 Bill Fair 与 Earl Isaac 于 1956 年创立，公司资料称其在 1958 年构建了早期信用评分系统。这一组织创新把统计模型、计算系统和业务决策结合成可持续交付的产品，而不是一次性的研究项目。FICO 公司资料

规模化同时把另一个问题推到台前：一致执行不必然等于公平、准确或可申诉。[事实] 1970 年的 FCRA 围绕征信信息的准确性、允许用途和消费者权利建立框架；ECOA 及其后续修订把受保护特征、公平审贷与“经验推导且统计上可靠”的评分系统放进制度讨论。FTC 对 FCRA 的历史回顾、1976 年 ECOA 修法文本

这一阶段形成了后来延续数十年的主流架构：模型团队维护相对稳定的风险排序，业务团队在其上配置准入 cut-off、额度档位和价格档位。这个“模型—政策分层”当时非常有效。模型不必随每一次资金成本或市场目标变化而重训，业务也能在受控边界内调整政策；验证、理由通知和变更管理因而有了相对清晰的责任对象。

它解决了批量自动审批、跨场景一致执行和初步可审计的问题，却把组织边界固化成了系统边界。风险模型、额度表、定价表和营销策略开始由不同团队维护，各自拥有自己的样本、目标和更新时间。一个模块可以准确预测违约，另一个模块可以准确预测响应，拼在一起却未必产生更好的合同。

[推断] 治理谱系在这一时期留下的并不是“必须使用简单模型”，而是更持久的原则：决策必须有明确用途、可复核依据和可追责流程。它留下的包袱则是把“可审计”等同于“模块彼此分离”。当循环信贷和信用卡进入大规模账户运营，开户时的一张静态分数再也不足以支撑贷中决策，行为状态谱系因此走上前台。

2.3 1980s—1990s 中期：行为评分把对象从“一个人”改成“一个正在变化的账户”

申请评分隐含了一个稳定世界：在申请时测量客户，随后依据这次测量管理较长时间。循环信贷打破了这个假设。余额、利用率、还款节奏、逾期、消费频次和现金提取不断产生新信息；同一个客户在稳定使用、突然加杠杆和刚刚恢复还款时，不应被视为同一种状态。

[事实] 芝加哥联储 1981 年工作论文已经记录信用评分在信用卡和零售场景的大规模采用，并提到大型零售机构按卡计划和门店维护大量模型，模型用于发卡、授权额度并随经济与信用条件重估。Federal Reserve Bank of Chicago, 1981 行业系统随后把行为分、分群动作和生产试验接到一起。FICO 在 2007 年 SEC 年报中将 TRIAD 描述为存量客户的 adaptive control 系统，覆盖风险管理、调额、定价调整、逾期管理与 champion/challenger；同一材料称其当时管理约全球 65% 的信用卡账户。FICO 2007 年 10-K

主流范式从“一次评分、长期使用”变成“周期性更新状态、按状态选择动作”。champion/challenger 又加入了一个关键制度：现行策略不再是不可挑战的真理，替代策略可以在受控流量中接受结果检验。现代决策系统最重要的组织原型已经出现——状态更新、action mapping、对照比较和结果反馈处在同一个运营循环里。

这个范式解决了贷中管理的时间问题。机构可以根据账户的新行为收缩或扩张暴露，也可以将策略迭代从大规模一次性切换改成受控比较。可是，当时的算力、数据和审计方式更适合“少量分群 × 少量动作”：行为分被切成档位，档位被映射到规则，额度和价格通常分开调整。局部试验能比较两套策略，却很少覆盖完整的 额度 × 价格 × 期限 组合。

更深的限制藏在反馈里。旧策略决定了谁得到什么额度与价格，客户随后表现出来的使用和风险又被拿来训练下一代模型。大量数据因此是“被政策选择后的数据”，并非自然世界中对所有动作都有观察。传统 champion/challenger 让系统开始学习，却常只保留最终 A_t，没有完整区分分配 R_t、展示/接受 D_t，也没有保存每个 candidate action 的选择概率。

[推断] 行为状态谱系把客户从静态类型改写为动态过程，但动作仍常被当成状态更新后的机械结果。合同经济学随后补上关键一课：额度和价格不只是根据风险被动设置的结果，它们本身会改变谁愿意借、借多少以及如何偿还。

2.4 1981 理论起点、1990s 中期—2008 加速：合同经济学让额度和价格从“报价结果”变成“行为干预”

当风险评分可以排序、行为评分可以更新，机构自然倾向于把合同设计简化为一条单调规则：风险越高，价格越高、额度越低。这个逻辑忽略了合同会反过来选择客户和改变行为。[事实] Stiglitz 与 Weiss 1981 年关于信贷配给的经典研究指出，利率上升会通过逆向选择改变申请人构成，也会通过激励效应改变借款后行为，因此贷款人的期望收益未必随利率单调上升。Stiglitz & Weiss, 1981

合同经济学由此改变了问题：价格不只是对既定风险收取的补偿，额度也不只是对既定风险设置的上限。两者共同决定客户是否接受、使用多少、承担多大还款压力，并进一步改变暴露和违约。风险定价在零售信贷扩张后，风险、资金成本和预期收益开始被放入共同的货币化框架；Basel II 又把 PD、LGD、EAD、内部评级和验证嵌入资本计量，使额度变化同时成为客户行为、预期损失与资本占用问题。Basel II 官方文件

后来的高质量实证把这种机制从理论可能变成可测量事实。额度随机试验和阈值准实验发现，提高可用额度会改变借款或消费，且响应随风险状态和流动性约束显著不同；价格随机试验也发现需求弹性会随人群和观察窗口变化。Aydin 额度随机试验、Agarwal 等额度阈值研究、Karlan 与 Zinman 长期价格试验

这些研究解决的是“动作会不会改变结果”和“平均效应是否异质”，并没有给出可跨产品、跨时期复制的调额比例、调价步长或观察周期。一个试验中的大幅额度冲击不能线性缩放为另一个产品的小幅提额；阈值附近的局部效应也不能直接外推到全客群。[推断] 历史在这里留下了一条重要证据纪律：经验估计可以证明合同动作值得建模，却不能替机构决定通用参数。

合同经济学让机构拥有风险—收益共同语言，也暴露了旧架构最深的裂缝。若额度模型、价格模型、风险模型和营销响应模型分别在历史政策数据上训练，每个模块都可能在自己的指标上准确，联合起来仍可能选择错误合同。价格改变接受概率，额度改变使用与 EAD，两者还会共同改变偿付压力；“先选最优额度，再选最优价格”只有在交互可忽略时才成立，而真实信贷很难普遍满足这一条件。

到危机前夕，系统已经知道合同会改变结果，却仍缺少一个可治理的方式把长期利润、偿付能力、资本和客户保护同时放入 action selection。风险事件与监管反思把这个缺口推到中心，下一阶段的主角不再只是更精细的定价，而是受约束的生命周期优化。

2.5 2009—2015：危机后治理把“为什么可以这样行动”写进优化问题

危机前的风险定价和自动化扩张证明机构能够更细地分层，却也暴露出局部模型准确并不足以保证组合安全。过度授信、还款能力不足、模型误用和不可解释的合同变化，使问题从“模型是否预测得准”转向“这个模型和策略为何适合此用途，谁批准，如何监测，失效时如何退出”。

[事实] 美国 CARD Act 及现行 Regulation Z 对信用卡发卡或提高额度前的偿付能力评估提出要求；2011 年的 SR 11-7 则曾系统化模型开发、使用、独立验证和治理。到本报告研究截点，SR 11-7 已被 2026 年发布的 SR 26-2 取代，后者继续强调按模型风险、机构规模和复杂度实施风险相称的验证、监测与治理，而非给出统一业务阈值。CFPB Regulation Z §1026.51、美联储 SR 26-2

在方法侧，利润优化和动态规划开始更直接地进入账户管理。[事实] So 与 Thomas 2011 年用马尔可夫决策过程研究信用卡账户管理，将行为评分带作为状态、额度作为动作，并以利润而非单一违约率比较静态与动态策略。EJOR 论文、预印本 这一范式把多期 state transition、未来收益和当前动作放进同一计算框架，承认今天的调额可能改变明天的账户。

它解决了两个长期缺口：经营目标不再只等于降低 PD，治理也不再只是模型外部的一次审批。偿付能力、资本、损失、消费者保护、人工覆盖和回滚开始成为动作能否进入生产的组成部分。一个优化器即使给出更高期望利润，如果依赖不可靠数据、越过合同边界或无法复现，也不是可接受策略。

当时没有直接跃迁到全自动联合决策，原因并不只是算力。违约和长期利润需要较长时间成熟，历史策略高度确定，未 executed action 缺少结果；多期优化依赖可信的 state transition，而额度和价格恰会改变这个转移；合规和模型风险团队也更容易验证评分卡与决策表，而非不断变化的黑盒策略。于是主流落地形成了“优化建议 + 规则执行 + champion/challenger 验证”的折中。

[推断] 这一折中保住了安全边界，却仍无法回答一个更尖锐的问题：历史数据显示某类客户在某个 offer 下表现较好，究竟是 offer 造成的，还是旧策略本来就挑中了更好的人？要把利润优化从结构假设推进到可辩护证据，因果识别谱系必须接管 action effect 的定义。

2.6 2015—2020：因果识别把问题从“谁会发生”改写成“动作让谁改变”

传统风险模型估计的是 P(违约 | C_t,S_t,历史政策)，响应模型估计的是 P(使用 | C_t,S_t,历史分配与 executed action)。两者都可以很好地预测已经发生的世界，却不自动回答同一账户在另一个动作下会怎样。旧策略又会主动选择客户：如果高额度长期只给优质账户，“高额度组违约更低”很可能是选择结果，而不是高额度降低了风险。

因果推断提供了不同的语言。potential outcomes 把 executed action A_t 取值为 a 与 baseline a0 时的结果写成 Y(A_t=a) 与 Y(A_t=a0)，action value 关心的是二者之差。Rubin 的 potential outcomes 框架奠定了现代识别表达；商业响应领域随后发展 differential response 与 uplift，提醒普通响应模型容易找到“本来就会响应的人”，而不是“因动作才改变的人”。Rubin 1974、Radcliffe & Surry 1999、Lo, True Lift

2010 年代中后期，因果树、因果森林、DML 与 observational policy learning 把异质性效应和高维状态带入可用方法栈。因果森林用于估计不同状态下的 treatment effect，DML 通过正交化与交叉拟合降低高维 nuisance model 的正则化偏差，policy learning 则进一步比较受预算、公平或可解释形式约束的分配规则。Causal Forest、Double/Debiased Machine Learning、Policy Learning with Observational Data

这次范式变化重写了“敏感度”。额度敏感或价格敏感不再是客户身上永久不变的标签，而是给定 C_t、S_t、action dose 与结果窗口时，对潜在状态 Z_t 的条件效应判断。当前合同 C_t 不能简单删除：它会影响需求、约束和未来 feasible action；但也不能让它直接代表下一次动作的价值，因为它由旧政策形成，带有内生性。正确的分工是把 C_t 与决策前 S_t 放入状态条件，再用随机、准实验或有明确假设的观察性设计识别 action effect。

action semantics 也必须在实验、日志和 policy 中保持一致。系统可以把 R_t 与实际 A_t 编码为目标合同，也可以编码为相对 C_t 的变化；同一个 estimand 和 policy version 内不应混用。若“提额一级”在不同账户上对应完全不同的经济剂量，却被记录成同一动作，估计出的 response surface 就失去清晰含义。D_t 还会使“被分配”“看到/接受”和“实际生效”分离：随机分配通常直接识别 R_t 的 ITT，若要解释实际 A_t 的效果，还需处理接受、执行与依从性选择。[推断] 当前合同 C_t 可以作为 policy 条件，用来决定哪些目标合同或相对变化可选；它本身既不是 treatment effect，也不能被解释成“当前价格导致了当前风险”的因果证据。

因果方法解决了“预测”与“干预”的概念混淆，却没有消灭识别条件。随机试验仍需处理执行偏差、失访、干扰和外推；RD 主要识别阈值附近的局部效应；DiD 依赖平行趋势等条件；DML 不能自动消除未观测混杂；DR 也不能恢复历史从未尝试过的动作。[事实] 当目标策略在某类账户选择的动作，旧策略在该类账户上从未选择时，overlap 缺失会使重要性权重不存在或极端不稳定。OPE overlap 研究

还有一个容易被忽略的断点：CATE 高不等于 policy value 高。一个动作可能提高使用，却同时提高损失、资本占用或客户退出；效应估计必须经过统一价值口径和硬约束，才能进入 action selection。[推断] 因果识别把“该对谁行动”变成了可研究问题，也暴露了数据基础设施的不足——没有 R_t、D_t、A_t、decision-time logged propensity、版本和 matured outcomes，方法越复杂，越容易把模型外推包装成因果结论。

这直接催生下一阶段：与其只训练一个更好的效应模型，不如把“candidate policy 能否超过现行策略”作为独立估计对象，并在安全边界内主动生产可识别数据。

2.7 2020—2026：OPE 与 constrained learning 把研究对象从 model 推进到 policy

champion/challenger 的优点是安全、清楚，缺点是每次只能比较少量人工设计策略；环境变化快时，固定矩阵迭代缓慢。contextual bandit 提供了更细的局部学习框架：给定决策时 C_t 与可观测状态 S_t，在多个 assigned action R_t 中选择一个，同时用 controlled exploration 更新对 action value 的不确定性。它并非突然出现的新思想。Epoch-Greedy、LinUCB 和基于随机日志的 replay evaluation 在 2007—2010 年已经建立关键方法基础。Epoch-Greedy、LinUCB、Replay Evaluation

真正推动其进入高风险决策讨论的，是 off-policy evaluation (OPE)与保守学习逐渐成熟。双重稳健 OPE 将 reward model 与 logging policy 信息结合，Counterfactual Risk Minimization 从带 propensity 的历史反馈中学习策略并控制估计方差；保守 contextual bandit 则要求新策略相对既有 baseline 维持最低表现保证，预算约束 bandit 把稀缺资源直接写入选择问题。Dudík 等双重稳健 OPE、Counterfactual Risk Minimization、Conservative Contextual Bandits、Contextual Bandits with Knapsacks

这套方法把系统推进了三步。评估对象从单个模型的 AUC、KS 或 Qini 转为整套策略相对 baseline 的价值；探索不再只是营销流量分配，而成为生产 counterfactual 证据的机制；资金、资本、风险、公平与客户保护也可以作为 action space 和约束进入策略，而不是等模型给出结果后再补一层人工拦截。

信贷场景也让它的边界格外清楚。delayed reward，未借款者没有还款标签，R_t 还要经过 D_t 才可能成为 A_t，公平和解释要求也不能被平均收益覆盖。如果一次动作对未来 C、S 与 Z 的影响在研究窗口内可以忽略，contextual bandit 可作为近似单步的安全局部学习：它回答“给定当前可见状态，下一次分配什么”，不是动态决策理论的终点。

若研究目标从定义起就跨越多次决策，且 A_t 会改变未来合同 C_{t+1}、可观测状态 S_{t+1}、潜在状态 Z_{t+1} 或下一次 feasible action，那么问题从定义起就是 dynamic treatment regime、MDP，或在 Z_t 不可完整观测时的 POMDP，而不是 bandit 运行后再“升级”出来的问题。候选动态策略也不能只靠单步 OPE 验收，需要 sequential OPE 或相应纵向因果估计处理多期动作—状态反馈。Dynamic DML、Sequential Doubly Robust OPE 只有当各期分配和接受机制、overlap、state transition 与 long-horizon reward 可以识别，日志又能复原 R_t→D_t→A_t 的完整轨迹时，这些动态框架才有成立基础；否则，以 bandit 做受控的局部近似，并用长期对照验证外溢影响，反而更诚实。

因此，“在线学习”在信用决策中首先是治理架构，而不是一个算法包。系统必须保存决策时 C_t、S_t、完整 feasible action set、R_t 的 logging propensity、D_t、实际 A_t、策略与随机种子版本；新策略需要现行 champion 作为 safe baseline，overlap 不足时要返回“不可识别”，异常时可以限流、冻结动作或回滚。复杂算法也不能替代具体动作理由。CFPB 对不利行动通知的解释明确指出，黑箱复杂度不能成为无法提供具体、准确理由的借口。CFPB Circular 2022-03

[推断] 截至 2026 年，公开研究足以证明信用定价、调额和 constrained policy learning“可以被建模和试验”，不足以证明全自动联合 bandit/RL 已成为大规模零售信贷的普遍生产范式。可辩护的结构仍是规则安全域、可识别实验、因果响应、policy evaluation 与保守探索逐层提供证据。任何探索分配、对照比例、action dose 或冷却周期都应由适用法律、风险偏好、统计功效、outcome maturity curve、overlap 和可恢复性校准，而不是从论文或同业案例中抄成“标准答案”。

这一阶段解决了静态策略难以持续适应的问题，也把历史矛盾推到最尖锐的位置：系统终于能边行动边学习，但它学习到什么，完全取决于 reward、约束、日志和探索如何设计。错误的目标函数配上更快的学习，只会更快地把短期收入偏好、选择偏差或不公平待遇固化进政策。

2.8 汇流后的结论：系统竞争力从 better score 转向 trustworthy decisioning process

到 2026 年，五条谱系可以被放进同一个决策表达。风险测量与行为状态共同定义当前合同 C_t、可观测状态 S_t 以及对潜在状态 Z_t 的 belief；合同经济学说明分配 R_t 会通过展示/接受 D_t 形成 executed action A_t，并改变需求、暴露和偿付；因果识别比较 Y(A_t=a) 与 baseline Y(A_t=a0)；policy learning 在合法、风险、资本、公平和解释约束内比较 long-term incremental value；governance 则保证 C_t/S_t/R_t/D_t/A_t、reason code、版本和结果能够被验证、replay 与回滚。动作 a 的语义必须固定为 target contract 或相对 C_t 的 change 之一。

选择策略 π，不是最大化某个模型分数，而是：

max  E[长期增量收入 - 增量信用损失 - 资金/资本/运营成本 + 可归因长期价值]
 s.t. 合法合同、偿付能力、风险暴露、公平、解释、overlap 与可回滚约束

contextual bandit 只承接 state transition 可忽略时的近似单步局部学习。只要目标跨期且 A_t 改变未来 C/S/Z，理论对象从定义起就是 DTR、MDP/POMDP 与 sequential OPE；没有可识别的转移、long-horizon reward 和轨迹支持，算法名称不能证明动态策略有效。

其中最关键的历史教训是：策略会制造自己的训练数据。 旧政策决定谁被分配什么合同，谁看到并接受，什么动作最终生效；这些环节又共同决定未来可见的使用和坏账。样本量增加并不自动增加 counterfactual 信息，确定性越强的旧策略，反而可能让替代动作越难识别。现代系统的稀缺能力因而不只是模型复杂度，而是一条持续生产可识别证据的数据供应链。

证据边界： 本章引用的随机试验、准实验和监管文件用于证明机制、异质性与治理方向。研究中的效应数值受市场、产品、时期、入组条件、action dose 和结果窗口限制；监管共同强调的也是风险相称的验证、监测、审计和消费者保护，而非跨市场统一的经营阈值。任何参数都应标明它是监管硬约束、研究估计、内部观测、经验默认还是待验证假设。

纵轴走到这里，横轴的问题也随之清楚：规则与评分卡、预测响应、uplift/CATE、OPE 和 bandit 并不是五选一的“代际竞赛”。它们分别承接历史上尚未消失的职责。下一章要比较的，不是谁最先进，而是谁能在整条决策链中回答哪一个问题、依赖什么证据，又必须被什么边界约束。

三、横轴：先比较 end-to-end decision paradigms，再拆 capability components

Rules、predictive models、CATE、OPE 和 bandit 经常被写成五条互相替代的“技术路线”，但这个分类混合了不同抽象层。Rules 是约束与执行机制，predictive models 和 CATE 是 outcome-estimation components，OPE 负责 policy evaluation，bandit 则同时包含 action selection 与 data acquisition。它们可以组合成多种完整系统，不能放在同一张算法先进度榜单上。

横轴需要先比较端到端范式：一个范式怎样定义状态和动作、凭什么比较 counterfactual、怎样求解策略、如何处理时间反馈与约束。完成这一层后，才有意义比较各能力组件分别负责哪一种不确定性。

规则策略和 predict-then-optimize 主要延续了评分与运筹传统；结构需求模型把信息经济学中的选择、逆向选择和合同激励写进模型。利率并非只补偿既定风险，它会改变申请者构成和借后行为（Stiglitz & Weiss）；结构化合同研究也表明，首付、额度或价格可以同时筛选风险并改变贷款规模（Einav, Jenkins & Levin）。结构范式因此比普通预测面拥有更强的 counterfactual 表达力，也承担更强的模型设定风险。

因果 policy learning、安全 bandit 与动态 regime 则分别解决固定窗口的策略选择、受控信息获取和跨期状态控制。它们也不是线性升级关系：若动作对未来状态影响可忽略，动态模型可能徒增不可验证假设；若状态反馈显著，单步 bandit 再精细也没有覆盖真正的问题。

3.2 统一 notation：current offer 是 state，assignment、acceptance 与 execution 必须拆开

为避免方法之间悄然更换 estimand，本报告固定以下符号：

  • C_t=(L_t,p_t,q_t)：决策前已经生效的当前合同，分别表示额度、价格/综合费率与期限；
  • S_t：决策时可观测的其余状态，包括风险、余额、利用、历史响应和市场信号；
  • Z_t：需求、流动性、偏好或竞争环境等不能被完整观测的潜在状态；
  • R_t：策略或随机机制分配的 assigned action；
  • D_t：提议是否被展示、理解或接受的中间环节，必要时可继续拆分；
  • A_t：真正执行并生效的动作；
  • Y_{t:t+h}：即时响应、使用、损失、成本、留存和后续状态组成的 outcome trajectory。

当前合同 C_t 属于决策前状态，策略可以条件化它；它不是本次干预。R_t 与 A_t 的动作编码必须在一个 estimand 内固定为两种形式之一：目标合同 C_t^*，或相对变化 ΔC_t=C_t^*−C_t。两种形式不能在训练、策略比较和执行时混用。采用相对变化时，feasible action set 和结果合同都必须由 C_t 显式推导；采用目标合同时，则要把路径依赖保留在 S_t 与历史中。

这组符号同时拆开三个问题：条件预测描述 E[Y_{t:t+h}|S_t,C_t,R_t,D_t,A_t]；action effect 比较同一决策前状态下的 potential outcomes，例如 E[Y_{t:t+h}(A_t=a)-Y_{t:t+h}(A_t=a_0)|S_t,C_t]；policy evaluation 的是分配规则 π(R_t|S_t,C_t) 经过展示、接受和实际生效之后的总体结果。随机分配识别的是 R_t 的 ITT 效应；若只研究真正生效的 A_t，还需要处理依从性和选择，不能把二者都叫“实验效应”。

3.4 Rules / scorecards：不是落后的 optimizer，而是 decision guardrails

规则/评分卡最擅长把偿付能力、风险偏好、合同限制和人工例外转成可重复决策。它能在分配 R_t 前逐账户剔除动作，而组合均值约束做不到这一点；固定 reason code、覆盖顺序和审批权限也便于重放。零售信贷监管手册长期把定价放在风险、竞争价格和业务量共同权衡的框架中，并提醒过高定价可能带来逆向选择或客户流失（OCC Retail Lending Handbook）。

它的边界同样明确。评分卡主要描述既有政策下的风险排序，无法仅凭历史相关性回答改变合同造成多少增量价值。规则层最合理的职责是依据 (S_t,C_t) 输出安全提议集合 A_safe(H_t)、每个被排除动作及理由，再由完整决策范式在集合内比较。规则由此从“猜最优动作的旧系统”变成“授予其他组件决策权限的可审计接口”。

3.5 predicted response surface：最好的地图，也可能不是因果地图

预测组件可以分别估计 R_t 被展示和接受的概率、实际 A_t、使用规模、损失与价值，对一维连续价格或额度尤其自然。它能画局部响应曲线、检查单调性、生成候选合同，也能充当 DML/DR 的结果与 propensity model。

问题在于，历史提议通常由旧策略分配。若低价或高额主要给了原本更稳定的人，条件预测会混合客户差异、旧政策判断与真实合同效应；优化器随后会偏爱旧策略最少涉足、预测却最不可验证的区域。贷款定价研究已经直接展示这类选择偏差（A Causal Perspective on Loan Pricing），连续价格还可能需要工具变量等额外识别设计（Personalized Pricing with Invalid Instrumental Variables）。

普通预测面是能力组件；只有当它与明确的经济行为结构、识别条件和约束求解器共同组成闭环时，才成为“结构需求模型 + 约束优化”范式。增加模型深度不能替代这个抽象层变化。

3.6 Uplift/CATE：从“谁会发生”跨到“谁因动作而改变”

对基准动作 a_0 和 candidate action a_1，CATE 估计：

τ_A(S_t,C_t)=E[Y_{t:t+h}(A_t=a_1)-Y_{t:t+h}(A_t=a_0) | S_t,C_t]

若研究的是策略分配，则应另写 τ_R；当展示、接受和实际生效不完全一致时，τ_R、τ_A 与 LATE 回答不同问题。这个区分比选择哪一种 meta learner 更基础。

因果森林、广义随机森林和 DML 为非线性异质性、高维 nuisance model 及部分连续处理提供工具（Causal Forest、Generalized Random Forests、Double/Debiased Machine Learning）。但 CATE 的可信度来自实验或识别设计，不来自算法名称。个体 counterfactual 没有可同时观测的真值；动作从二元扩到多臂乃至二维连续空间后，每个“状态 × 动作”单元的有效样本会迅速稀释。

它的最佳生态位是少量、经济含义明确的联合合同：先估计各动作相对基准的效应与不确定性，再把稳定异质性转为受限 policy class 或浅策略树（policytree）。对支持不足的区域，保留 baseline 比给出虚假个性化更符合因果证据。

3.7 DR/OPE：它比较策略，不是再训练一个响应模型

CATE 说明局部动作可能有效之后，仍需回答：把这些效应组合为一套分配策略 π(R_t|H_t)，总体是否优于 champion？OPE 正是策略比较组件。Direct Method 依赖 outcome model，IPS 依赖 logging policy 的 logged propensity，Doubly Robust 将二者结合；在相应正则条件和 overlap 下，outcome model 或 logging policy 模型其中一个正确即可获得一致估计（Doubly Robust Policy Evaluation and Learning）。

“双重稳健”不会创造缺失的 counterfactual。旧策略从未分配的动作没有支持；事后拟合 propensity 不等于恢复决策时真实概率；若 R_t 与 A_t 因展示和接受而分离，忽略 D_t 还会把分配 policy value 与实际 treatment effect 混在一起。新策略远离 logging policy 时，少数轨迹获得巨大权重，点估计可能由极小有效样本量支撑。common support 缺失会使标准 OPE 严重失效（OPE under Deficient Support）。

策略采用条件因此不应只是一项期望价值：还需同时给出相对 baseline 的置信区间、有效样本量、权重分布、状态—动作覆盖、结果成熟度和分群稳健性。无支持区域应保持 baseline、补充实验或缩小候选 policy，而不是继续更换估计器。

3.8 Constrained contextual bandit：第一产物不是 personalization，而是可信的 logged data

Contextual bandit 根据 (S_t,C_t) 在 safe action set 中分配 R_t，只观察经 D_t 后实际生效的 A_t 与局部结果。它相比确定性规则最重要的新增能力，是前瞻记录已知分配概率，让未来日志具备可评估性。Vowpal Wabbit 的标准工作流也把 context、action、选择概率和 reward 作为不可分割的日志要素（VW Contextual Bandits）。

“受约束”至少包含三层：逐笔禁止动作必须在分配前剔除；资金和资本等资源约束需要累计控制；新策略相对 baseline 的表现要有保守边界。保守 contextual bandit 和带预算的 bandit 分别提供后两类理论工具（Conservative Contextual Linear Bandits、Linear Contextual Bandits with Knapsacks），但它们不自动等价于逐笔合法、公平或可解释。

它应被明确定位为安全域内、近似单步或短反馈的局部学习组件，不是动态理论终点。若本期合同显著改变下一期 S_{t+1}、C_{t+1}、Z_{t+1} 和后续可选动作，问题应转向 dynamic treatment regime、MDP 或 POMDP，并以 sequential OPE 比较整条策略轨迹；序列双重稳健估计提供了相应框架（Doubly Robust Off-policy Value Evaluation for Reinforcement Learning）。POMDP 的 belief state 可以表达对 Z_t 的后验不确定性，却不能证明旧策略没有使用模型未见的信息；隐藏混杂仍需独立的因果识别证据。

3.9 组件怎样连接：按证据分配权限，按状态反馈选择范式

在近似单期问题中，规则组件先生成 safe action set；预测与 CATE 组件估计候选合同的结果、增量和不确定性；DR/OPE 比较完整 policy；只有在概率日志、延迟结果和 fallback baseline 均可靠时，bandit 才获得受控信息获取权限。结构需求范式可以复用预测组件，因果 policy learning 可以复用 CATE 与 OPE，安全 bandit 也需要规则、outcome model 和 OPE。组件可以共存，端到端范式却必须明确说明自己依赖哪一种识别逻辑。

若动作造成显著跨期状态反馈，静态组件仍可用于风险估计或局部效应，但决策对象已经从单次动作变为策略轨迹。此时需要动态 regime/MDP/POMDP 与序列策略比较，而不是把远期结果简单塞进单步 reward。模型治理也应随角色改变：规则审计覆盖与理由，预测审计校准和外推，CATE 审计 estimand、overlap 与敏感性，OPE 审计真实 propensity、支持和有效样本量，动态策略还要审计转移、观测模型与轨迹支持。复杂算法不能成为无法说明具体决策原因的理由；相关原则可参见 CFPB Circular 2022-03 与 Federal Reserve SR 26-2。

四、横纵交叉：六个由历史与方法版图共同推出的非显然结论

纵轴显示，信用决策从风险评分、行为账户管理、风险定价，一路走到因果效应与动态策略；横轴则说明，今天并不存在一个能够取代前代能力的“终极模型”。两条轴交叉后，关键变化不是算法越来越复杂，而是系统必须重新安排证据、权限、action semantics 和反馈时钟。

4.1 过去最成功的制度资产，正在制造今天最难偿还的因果债务

评分工业化和规则一致性曾解决大规模授信最棘手的问题：同类状态得到可重复、可审计的处理。它们越成功，R_t 就越确定；分配越确定，历史数据中可比较的替代动作越少。于是出现一个悖论：数据规模持续增长，counterfactual 信息却未同步增长。百万条由同一规则产生的记录，可能不如一小批分层随机日志更能回答“换一个合同会怎样”。

风险评分的职责因此需要重新定位。它仍是 safe action set 和价值函数的关键输入，却不应独自承担合同选择。动作日志也不是普通数据治理，而是偿还因果债务的会计系统：除 S_t,C_t 和最终 A_t 外，还要保存全部 candidate action、分配 R_t、展示/接受 D_t、真实概率、剔除理由、override 与版本。规则由此保留历史形成的审计优势，也为后续识别留下空间。

4.2 “敏感度”不是隐藏的人格标签，而是状态、合同与动作共同定义的响应

行为评分把对象从静态借款人改成动态账户，因果方法又把问题从“谁会响应”改成“哪个动作使结果改变”。两者交叉后，长期固定的“价格敏感度=0.8”失去清晰含义：同一个人会因 C_t、余额、需求状态、竞品环境和目标合同不同而表现出不同响应。

更合理的输出不是一个标签，而是条件 action value 表：给定 (S_t,C_t)，分别报告候选目标合同或相对变化的预期增量、不确定性、common support 和可执行性。Z_t 可以用 belief state 表达为后验分布，但它只解决部分可观测状态的表示问题。若旧策略依据当前数据中缺失的信息分配 R_t，隐藏混杂仍然存在；更复杂的状态表示不会把观察性关联变成 action effect。

4.3 方法升级不是给 model 更多 authority；每个 decision claim 都要跨过更高的 evidence bar

纵轴容易给人线性升级的印象：评分卡之后是机器学习，机器学习之后是因果，再走到 bandit 和动态控制。端到端范式表却揭示了相反的治理逻辑：不同范式解决不同反馈结构，并不存在一条所有机构都应走完的阶梯。

从能力组件看，预测模型只应获得生成候选和描述结果的权限；CATE 在识别成立的区域获得比较动作的权限；OPE 在支持充分时提供采用 candidate policy 的证据；bandit 只有在 safe action set、baseline 和延迟反馈可控时才获得局部探索权限；动态 regime 还需额外证明轨迹支持与转移假设。技术成熟度不等于模型复杂度，而是每个结论只触发与其证据强度相称的行动。

4.4 Action space 不是预先给定的菜单，而是一项统计设计

历史上的离散额度表和价格档常被视为落后遗产，连续优化则被视为自然终点。横轴比较给出更谨慎的判断：动作颗粒度越细，商业表达越灵活，但每个“状态 × 动作”单元的支持越薄，因果识别、OPE 方差、客户解释和运营复现都会变难。二维连续联动不是把两个连续模型接起来，它会成倍放大可识别性问题。

早期最好的 action space 往往不是最细，而是信息效率最高：把额度、价格、期限作为联合合同，设计少量经济含义清楚、彼此可比较的目标 offer，或固定版本的相对变化，并保留 baseline 与接近 baseline 的探索档。动作数量应由预期决策价值、样本支持、风险差异和解释成本共同决定。减少动作有时不是牺牲个性化，而是在用高质量证据换取可持续的个性化。

4.5 Outcome delay 决定算法上限；先定义 long-term value，再谈 online learning

行为评分时代留下了快速更新账户状态的能力，bandit 又把快速反馈变成优势；但信用损失、资本占用和关系价值的成熟速度远慢于展示、接受或使用。如果目标由最先到达的指标定义，系统会把“可快速观测”误当成“最重要”，并在 matured outcomes 出现前持续放大同一方向。

因此，系统需要双时钟：快时钟只负责流量、响应、异常和代理风险，拥有暂停权而没有扩大长期策略权；慢时钟用 matured outcomes 重算 policy value、验证代理指标并决定是否扩大策略权限。若 A_t 还会改变未来 S_t,C_t,Z_t 和可选动作，单步 bandit 的理论边界已经到达；此时需要动态 regime、MDP/POMDP 和序列 OPE。在没有可信 state transition 证据前，受限的短视策略可能比未经验证的长期 RL 更理性。

4.6 最稀缺的能力不是某个模型，而是把治理变成可学习接口

早期模型—规则分层的价值在于组织清晰，代价是风险、额度、价格和营销各自优化。今天若只把多个模型放到同一个服务中，烟囱仍然存在，只是接口从表格变成 API。真正的联动要求统一四个对象：决策前的 S_t,C_t，分配—展示/接受—生效链 R_t,D_t,A_t，跨时间结果归因，以及潜在状态 Z_t 的不确定性表示。

治理在这个结构中不只是限制。safe action set 缩小搜索空间；原因码定义可审计 action semantics；稳定对照和 logged propensity 使 OPE 成为可能；fallback baseline 让受控探索获得生存条件；固定目标合同或相对变化的编码，使跨期重放不会悄然更换 estimand。历史上为控制风险建立的制度资产，经重新设计后会成为持续学习的基础设施。系统竞争力最终来自一条可证伪链：每次分配都能重放，每项结果都能追溯，每个策略都能与 baseline 比较，每个结论都可能被后续成熟数据推翻。这条链也允许系统明确说“不知道”：证据不足时保留 baseline，而不是用模型自信替代识别条件。

六、没有 A/B 时，怎样从相关性走向可辩护的因果证据

6.1 先承认代理特征的正确用途

历史利用率、借满率、还款后复借间隔、看价不借、有效价格差等 proxy 很有用，它们可以稳定描述“当前约束状态”，也可用于分层实验和 candidate action 生成；但在旧策略非随机时，它们不能天然等于提额或降价的因果效应。

6.3 Time-varying treatment：区分 treatment–confounder feedback、washout 与 cooldown

连续调额/调价会形成 treatment–confounder feedback：上一次 action 改变下一次 state，而下一次 state 又影响后续 action。解决路径有两类：

  • 使用动态处理、g-method 或多期模型明确时序；
  • 在单次效应研究中用 washout 定义因果污染窗口，并用 cooldown 控制运营中的重复 action。两者目的不同，不能共用一个含混的“冷静期”。

washout 与 cooldown 的长度都应由 response maturity、risk maturity、合同边界和 contamination diagnostics 决定，而不是从固定天数模板抄取。紧急风险 action 可以保留 override，但必须记录 reason code。

七、通用理论设计：受约束的动态因果决策系统

7.1 Structural Causal Model (SCM)：先区分 state、action 与 outcome

一个通用的额度—定价系统至少包含六类变量：

  • X_t：决策时可观测的外生或慢变特征；
  • C_t：决策时已经生效的当前合同/offer；
  • S_t：其余可观测状态，包括风险度量、余额、历史响应和市场信号；
  • Z_t：不可完全观测的需求、流动性、偏好或竞争状态；
  • A_t：本次实际生效的动作，固定编码为目标合同或相对变化；
  • Y_{t+h}：在不同时间窗成熟的使用、收入、损失、留存和客户结果。

其动态关系可以抽象为：

A_t ~ π_b(a | H_t)                                  历史/当前策略选择动作
Y_{t+h} = g_h(S_t,C_t,Z_t,A_t,M_t,ε_{t+h})          动作产生多时间窗结果
Z_{t+1} ~ P_Z(· | Z_t,S_t,C_t,A_t,M_{t+1})          潜在 state transition
(S_{t+1},C_{t+1}) ~ P_O(· | S_t,C_t,Z_{t+1},A_t)    新观测与新合同形成

其中 H_t 是截至决策时点的历史，M_t 是可观测市场或宏观环境，ε_t 是新的随机冲击。这个表示法直接解释了 offer 内生性：current offer 是过去动作的结果和本次决策的条件；策略可以根据它选择动作，但不能把 current offer 与结果的历史相关性解释为本次动作的效果。 因而“把 offer 删掉”和“用 offer 的预测系数冒充干预效应”都不成立。

这里还需要区分三个容易被同一个字段名混淆的量：

理论角色 例子 可以进入哪里 主要风险
决策前状态 当前额度、当前价格、余额、最近一次变更 状态表示、资格与分层 遗漏会造成状态不充分
本次动作 目标合同，或维持/额度变化/价格变化的版本化相对编码 treatment/action、策略输出 表示法混用会改变 estimand
决策后变量 新利用率、实际借款、还款、后续 offer 结果或下一期状态 提前入模会造成泄漏和碰撞偏差

一个合格的理论定义不应只写“用当前额度预测需求”，而应把决策事件写成：E_t=(H_t,A_safe,R_t,D_t,A_t,π_b,Y_{t:t+h})。其中 R_t 是分配，D_t 是展示/接受，A_t 是 executed action，A_safe 是当时允许的 action set，π_b 是产生分配的 logged propensity。这个定义把“世界在决策前是什么样”“系统可以做什么”“系统分配了什么”“客户接受且实际生效了什么”“后来发生了什么”分开。没有这种时间语义，再高级的算法也无法判断一个特征是在动作之前、之中还是之后产生。

状态也不是越多越好。理论上的充分状态要保留影响动作与未来结果的共同原因，但应排除动作后的中介、未来信息以及仅在执行后生成的派生字段。若完整历史过长，可以用递归状态、序列表示或 belief state 压缩；压缩是否成立，取决于它能否支持结果预测、action selection 和 state transition，而不是取决于表示维度是否足够大。

7.2 Partial observability：敏感度更接近 belief state

真实需求、流动性紧张、竞品报价和心理偏好通常不可完整观测。理论上，这更接近 POMDP，而不是完全可观测的静态分类：

b_t(z) = P(Z_t=z | H_t)

Transition:  T(z' | z, S_t, C_t, A_t)
Observation: O(S_{t+1} | z', A_t)
Reward:      r(S_t, C_t, z, A_t)

b_{t+1}(z') ∝ O(S_{t+1}|z',A_t)
               · Σ_z T(z'|z,S_t,C_t,A_t)b_t(z)

b_t 是对潜在状态 Z_t 的 posterior belief，T 是 state transition，O 是观测生成机制。新的申请、查看价格、借满、提前还款或市场信号会更新 b_t；系统随后比较不同动作在该分布下的期望价值和下行风险。

[推断] 因此“价格敏感用户”不是本体论类别，而是给定历史和环境下的后验判断。理论输出应包含不确定性，并允许“信息不足/不可识别”，而不是强制给每个人一个稳定标签。

但需要把两个问题严格分开：POMDP 处理的是控制中的部分可观测性，因果识别处理的是历史动作与 potential outcomes 为何可比较。belief state 可以概括模型所见历史，却不能证明旧策略没有使用模型未见的信息。若存在隐藏混杂，belief update 不会把观察性关联自动变成 action effect；仍需随机化、可信准实验、工具变量、负向控制、敏感性分析或部分识别等独立证据。

原问题中三类“敏感度变化”可用同一状态空间解释：一是机构改变 offer，使约束本身变化；二是客户潜在需求或流动性 state transition；三是竞争环境改变相对价格与可替代选择。观察到的响应是三者共同作用的结果：

Observed Response
= f(潜在偏好与需求, 当前约束, 相对市场位置, action dose, 随机冲击)

因此，理论上更稳妥的对象不是一个静态“敏感度分数”，而是条件 response surface 或后验 action value 分布。对同一客户，τ_t(a,a0) 会随 S_t 和 M_t 改变；在数据稀疏时，它甚至只应被部分识别为一个区间。模型如果把一次降价后的恢复响应永久归结为“不敏感”，本质上是把状态变化误当成偏好不变。

belief state 还带来一个重要区分：动作既有控制价值，也可能有信息价值。一个近似等价的小剂量动作，即使即时收益不是最高，也可能帮助缩小需求状态的不确定性。但信息价值不能凌驾于客户保护和硬约束；探索只在预先定义的安全等价类内有意义。

7.3 Constrained objective：hard constraints 先定义 feasible set，再优化 long-term value

通用目标可以写为：

max_π  E_π[ Σ_t γ^t · r(S_t,A_t) ]

subject to:
  A_t ∈ A_legal(S_t) ∩ A_contract(S_t) ∩ A_risk(S_t)
  PortfolioRisk(π) ≤ R_max
  Exposure(π) ≤ E_max
  CustomerHarm(π) ≤ H_max
  FairnessGap(π) ≤ F_max
  ChangeFrequency(π) ≤ C_max

这里必须区分两类约束：

  • 硬约束先于优化器剔除动作，不能用拉格朗日罚项“付费违反”；
  • 软约束/组合预算可以进入优化或 policy selection，但需要置信界、压力测试和事后监控。

仅最大化平均利润仍可能把风险集中到尾部。理论设计应同时报告均值、分位数、CVaR/尾部损失、最大暴露和分群客户影响。

reward 本身也应是多时间尺度的，而不是把所有结果粗暴压成一个即时标签。设 r_0 为即时接受与使用，r_1 为近期收入和成本，r_2 为成熟信用损失，r_3 为后续留存、投诉与退出，则 policy value 可写成：

V^π(s) = E_π[r_0 + γ_1 r_1 + γ_2 r_2 + γ_3 r_3 | S_t=s]

折现系数不是纯技术参数，它表达不同利益和时间窗口的制度性权衡。若成熟损失尚不可见，不能用短期转化替代后宣称已优化长期价值；更诚实的输出是“短期价值可估、长期价值待成熟”，并给出上下界或压力情景。

不同约束也不能都被一个惩罚系数吸收。合法性、合同有效性、不得歧视等要求通常具有词典序上的优先级：先满足，再优化。组合风险预算、资金占用或探索成本才适合做拉格朗日约束或受约束策略选择。否则一个足够大的预期利润会在数学上“购买”违法或不可接受的动作，这与治理语义冲突。

7.4 Joint action：用 constrained action graph 管住 interaction、hysteresis 与 fallback

只有当目标函数可以按额度与价格分解、feasible set 也不存在共享约束或交叉边界，并且任一动作不会改变另一维的后续响应时，顺序优化才可能等价于联合优化。现实中二者共同影响接受、使用量、EAD、还款压力和留存，还共享偿付能力、暴露与合同约束，因此通常同时存在目标交互与约束耦合：

V(L,p | S) ≠ V_L(L | S) + V_p(p | S)

理论上可使用连续动作和 response surface；实践可识别性却由日志支持决定。一个稳健的通用原则是：先定义经济上合理的有限 joint action 集，估计主效应与交互效应；只有在局部支持充分、单调性和光滑性可辩护时，才加密为连续优化。

动作离散化不是算法落后，而是同时控制统计方差、安全、解释和实验复杂度。action grid 应由可识别性与决策价值共同决定，而不是由模型能输出多少小数位决定。

joint action 还意味着“最优价格”和“最优额度”未必各自存在稳定排序。若提额使客户从额度约束转为价格约束，降价的边际效果会改变；反之，价格下降带来的需求增长又会改变额度暴露和损失分布。可分离模型忽略的正是这个交互 state transition。

理论上可以把 action space 视为一个受约束的格：每个节点是一组合同条件，边表示一次允许的变化。策略不是在连续平面任意跳跃，而是在合法且可识别的边上移动。该表示自然支持三种机制：

  • 邻域约束：只比较有历史或实验支持的相近动作；
  • hysteresis 与冷却：进入与退出某一档位使用不同门槛，避免噪声造成往复震荡；
  • 不变基准：信息不足、边界冲突或异常时回到已验证的 baseline。

这也是控制理论与因果决策的交汇处：因果模型回答“一次改变会发生什么”，稳定性机制回答“连续重复改变会不会振荡或发散”。只具备其中一个，仍不是完整的动态系统。

7.5 Identification principle：没有 counterfactual variation，就没有 action value

任何识别路径首先都需要共同基础：干预及其版本定义清楚，time zero、目标总体和结果窗口明确，一致性可以辩护，干扰结构被限定，并且目标 estimand 所需的状态—动作区域有相应支持。在此基础上，随机实验、条件交换性、RD、IV、DiD 等才是不同的识别路径；它们识别不同总体中的不同效应，各自还需要附加假设，不能被写成可以任意互换的“几选一”。

共同基础与路径特有条件可以分开写：

  1. 干预明确与一致性：分配、展示、接受、实际生效的动作及其版本、剂量和执行方式不可混用；
  2. 目标与时间明确：先定义资格总体、time zero、结果窗口、删失和后续动作，再定义 potential outcomes；
  3. 识别路径明确：随机化依赖正确分配与执行；观察性 g-method/DML 依赖给定历史后的顺序交换性；RD、IV、DiD 分别依赖连续性、排除限制、平行趋势等路径特有假设；
  4. 支持与可迁移范围明确：目标策略可能选择的动作必须在相应状态中可比较，局部效应不能自动外推为全局 policy value；
  5. 干扰结构明确：资金预算、竞品和群体传播若让一个主体的动作影响其他主体结果，应进入网络、市场均衡或组合约束模型。

这些不是模型可以从数据中自动学出的结论，而是研究设计的前提。尤其是 propensity 只能校正已观测的 action selection 机制；若旧策略在某状态永远不给高额度，任何重加权都无法创造该动作的 counterfactual 结果。

对动态系统还要额外处理：

  • 当前动作会不会改变下一期混杂变量；
  • 后续动作是否污染本次结果；
  • 客户之间是否因预算、市场或竞争发生干扰；
  • 未接受 offer 的结果如何定义；
  • 延迟、删失和提前结清是否信息性。

[推断] 因果假设通常不能仅靠样本完全检验。当它们无法由设计、制度知识、诊断和反证分析合理辩护时，正确的理论输出是部分识别、上下界、敏感性分析或“需要新证据”，而不是用更复杂模型制造点估计。

还必须先定义 estimand，再选择算法。同一个“降价效果”可以指 assigned action 的 ITT、在额外依从性假设下的实际执行效应、以随机分配或阈值为工具识别的 LATE、历史 common support 内的 policy value，或某一动态制度下的长期价值。它们针对的人群、动作和 counterfactual 不同，不能仅因单位相同就互相替代。理论报告若不写 estimand，模型精度越高，结论反而可能越精确地回答错误问题。

7.6 Policy value 与 model performance 必须分别验证

状态模型、outcome model、propensity 模型、CATE 模型和 policy 都有不同目标。AUC、校准、Qini、CATE 稳定或 OPE value 不能互相替代：

对象 主要问题 理论验收
状态/风险模型 描述当前世界是否准确 校准、稳定性、分群误差
outcome model 给定动作下结果是否可预测 跨动作残差、外推诊断
Propensity/logging policy 历史动作概率是否正确 与决策日志逐笔一致、overlap
CATE/剂量响应 action contrast 是否可信 识别设计、异质性稳定、敏感性
Policy 选择动作是否提升总体价值 OPE、置信区间、约束、baseline 对比
端到端系统 组合使用是否安全可复现 回放、压力、熔断、版本和审计

7.7 前置能力与 problem class 不是一条成熟度阶梯

可审计语义、可识别数据、counterfactual 估计、policy evaluation 和安全控制确实构成能力前提；但 contextual bandit 与 MDP/POMDP 不是同一条“成熟度阶梯”的前后级，而是由问题结构决定的不同模型类：

问题结构 合适的理论对象 关键评估
一次性动作，或长期影响可可靠封装进单次 reward CATE、静态 policy learning、单步 OPE treatment effect、policy value、common support
重复决策，但动作不显著改变下一期状态 非平稳/受约束 contextual bandit regret、约束违反、单步 OPE
动作改变余额、风险、需求或未来可选动作 dynamic treatment regime、MDP/POMDP g-method、序列 DR/OPE、长期价值
多主体或组合预算产生相互影响 受约束 MDP、组合优化、博弈/均衡模型 组合风险、干扰、资源影子价格

工程上暂时采用静态近似，不等于理论问题已经退化为 bandit；它只表示对状态反馈做了受限近似，并应说明误差可能指向哪里。反过来，使用 RL 名称也不证明动态问题已被识别：没有 state transition 证据、序列支持和成熟长期 reward，序列模型仍可能只是在拟合历史路径。

因此，任何复杂方法都不能补偿基础缺失。没有可识别数据，bandit 只是自动化探索风险；没有硬约束，constrained optimizer 也不等于逐笔合法；没有成熟长期 reward，动态学习只会追逐代理指标。

7.8 一套 minimal closed-loop blueprint

不依赖任何特定机构，一个额度—定价联动系统至少应回答以下八个设计问题：

  1. 决策单位是什么：客户、账户、合同还是一次申请，跨单位如何处理；
  2. 状态边界是什么：哪些变量在 time zero 前可用，潜在状态如何表示；
  3. action semantics 是什么：维持、变化量、绝对合同和执行失败如何编码；
  4. 价值与约束是什么：多期 reward、硬约束、组合预算和客户结果如何分层；
  5. counterfactual 从哪里来：随机化、准实验、观察性假设或安全探索各覆盖哪里；
  6. 策略怎样被评估：个体效应、总体 policy value、不确定性和尾部风险如何分开报告；
  7. 系统怎样保持稳定：baseline、hysteresis、冷却、common support 回退和异常冻结如何定义；
  8. 知识怎样更新：新结果如何改变状态估计、action value 和 policy version，哪些变化需要重新识别。

这八项共同构成理论上的最小闭环。它们不预设必须使用评分卡、因果森林、DR 还是 bandit；算法只是实现其中某一推断或选择步骤的工具。一个简单方法若八项定义清楚，通常比一个复杂方法但状态、动作和 counterfactual 含混的系统更接近正确设计。

八、什么必须通用，什么必须 context-specific：design invariants 与治理边界

8.1 应标准化的是 semantics 与 proof obligations，不是 numeric defaults

通用理论能够给出若干不变量，却不能给出跨产品、跨法域的“神奇参数”。两类对象必须分开：

理论不变量 为什么不可省略
time zero、目标总体、动作版本与结果窗口明确 否则 potential outcomes 与 estimand 没有稳定含义
当前合同属于决策前条件，目标合同或相对变化二选一 否则状态、动作与执行结果互相污染
硬约束先定义 feasible set，优化器只在域内选优 否则平均收益可能补偿不可接受动作
分配、展示、接受、生效和后续使用分开 否则 ITT、依从效应与 policy value 被混用
logged propensity、common support 与版本可追溯 否则重加权、OPE 与 policy replay 没有依据
短期结果与成熟长期结果分层 否则快速代理指标会替代真正目标
不可识别、不可执行与 baseline 是合法输出 否则模型会在证据空白区强制排序

而观察周期、action grid、锁定或冷却长度、实验分配、探索预算、漂移诊断线、停止边界和 reward 权重都属于 context-specific design parameters。它们应由适用法律、目标 estimand、最小可检测效应、outcome maturity curve、最大可承受损失、state transition 速度、overlap 和不确定性共同校准。所审阅材料中的具体数字只能作为历史方案的经验假设，不能从其出现本身推出普适性。

8.2 一个可接受的 policy 需要满足五类 proof obligations

  1. 适用性证明：动作、数据、合同与目标总体符合相应的法律、权利和制度边界；
  2. 识别证明：estimand、time zero、分配与执行、混杂、支持、干扰和删失假设能够被清楚陈述并合理辩护；
  3. 估计证明：结果、propensity、CATE 或转移模型在其声称的范围内具有校准、不确定性与敏感性证据；
  4. 策略证明：候选 policy 相对 baseline 的价值、尾部风险、组合约束与不支持区域均被显式表达；
  5. 动态证明：若动作改变未来状态，整条轨迹的 state transition、序列支持、长期 reward 与 sequential OPE 必须成立。

这五类证明不是项目阶段，而是逻辑依赖。一个策略可以在预测上很强、在识别上很弱；也可以在平均价值上为正、在尾部风险或逐笔权利上不可接受。任何一类缺失，都限制它可以提出的 decision claim。

8.3 Governance 的核心是 falsifiability

治理的本质不是附加一张审批表，而是使系统的核心命题可以被反驳。最低限度应能回答：当时看到了什么状态；哪些动作可选；为何选中该动作；分配与实际生效是否一致；结果何时成熟；candidate policy 若替换 baseline 会怎样；哪些区域没有支持；新证据是否推翻原结论。

因此，监测也应对应不同命题，而不是依赖单一阈值：合法性和合同命题接受逐事件检验；执行命题检验分配—展示—接受—生效链；风险与价值命题同时检验均值、金额暴露与尾部；模型命题检验校准、漂移和外推；因果与策略命题检验 overlap、ESS、权重、不确定性和与对照的可重复差异。独立治理不能由同一个 reward optimizer 自我认证，因为优化器没有权限重新定义自身的约束和证伪标准。

九、行业实践案例与实验范式：先定 estimand，再选 assignment scheme

实验不是“有 A/B / 没 A/B”的二元选项，而是一组为不同 counterfactual gap 服务的研究设计。真正需要先回答的是：要识别哪个 estimand；treatment 是一次 offer、实际生效合同，还是连续 policy；随机化落在账户、门店、地区还是时间窗；客户不接受、动作 carryover、资源竞争和主体间 interference 怎样进入分析。只有这四件事明确，算法和样本量计算才有意义。

下面分两类列举。第一类是公开可核验的信贷或银行案例，可以说明金融问题曾怎样被实验化；第二类来自科技平台与医疗决策研究，只是可迁移的 design pattern，不代表它们已成为信贷行业的普遍生产实践。

Individual RCT / persistent holdout Aydin 与合作银行对 45,307 个既有账户随机提供 credit-line expansion（AER） 提额 offer 的 ITT，以及预设分群内的异质性 随机分配未被后续规则系统性覆盖；结果窗口和 attrition 清楚 适合一次性、离散合同动作；不能把某个大剂量试验结果线性缩放为任意 step size

9.4 可复用的 Experiment Card

每个试验在执行前至少应冻结一张 versioned Experiment Card：

  1. Decision question：本次只回答一个可证伪的问题，不把“优化全部经营目标”写成 hypothesis；
  2. Estimand：明确 ITT、per-protocol、LATE、CATE、policy value 或 dynamic regime value；
  3. Unit & interference：写清 account / customer / cluster / market-time unit，以及 spillover 和 shared-budget 结构；
  4. Treatment version：额度、价格、期限、展示方式、目标合同或相对变化必须可 replay；
  5. Assignment mechanism：记录完整 candidate set、strata、probability、seed、override 和 actual execution；
  6. Outcome clock：区分 proximal outcome、matured credit outcome 与 long-term value，说明 censoring 和后续 action contamination；
  7. Safety & stopping：hard constraints、exposure cap、sequential monitoring、pause rule、fallback baseline 和 long-term holdout 分层定义；
  8. Analysis contract：主分析、multiple testing、missingness、non-compliance、heterogeneity、sensitivity analysis 与外推边界预先写明。

[推断] 最成熟的实验范式不是名称最复杂的那个，而是能用最弱、最透明的假设回答一个边界清楚的问题。随机化可以提高 internal validity，但不会自动保证执行一致、长期结果成熟、主体间无干扰或能跨市场外推；准实验可以利用已有制度，但 local estimand 与 identification assumptions 必须写在结论旁边；adaptive design 可以提高信息效率，但不能取消 baseline、holdout 和 governance。

十、三种理论情景及其触发条件

10.1 基准情景：rule-based guardrails + CATE + OPE 的 hybrid architecture

[推断] 当长期结果延迟、替代动作支持有限、逐笔约束严格而 state transition 仍难可靠识别时，最自洽的形态是混合架构：规则定义安全域，因果响应比较少量动作，OPE 比较候选 policy，bandit 只承接近似单步、反馈相对快的局部学习。若未来能够稳定识别多期转移、序列支持与长期 reward，这个情景才会被更完整的动态 regime 取代。

10.2 最危险：用“AI 个性化”包装相关性定价和短期 GMV 优化

系统用 current offer 和短期行为预测下一动作，却没有实验、candidate action 概率和长期损失；降价激活后被判“不敏感”，再涨价造成震荡；或提额增加短期 GMV，却把金额暴露集中到坏客户。模型离线指标可能持续变好，但组合真实价值恶化。

可证伪信号包括：预测或 uplift 指标上升但随机对照/DR 价值不一致；candidate action 覆盖缩窄；平均风险平稳而损失严重度、金额暴露或尾部风险上升；人工覆盖增多；reason code 与实际约束不一致。

10.3 理想情景：action semantics 成为共享基础设施，policy 可被持续证伪

系统把“每次决策能否重放、每个动作能否识别、每个参数能否追溯”作为核心性质。知识从少量联合实验、可解释 policy 与长期对照逐步积累；任何区域支持不足都显式显示“不知道”。该情景成立的可观察条件是：替代动作拥有真实概率、长期结果能回填到同一决策事件、candidate policy 可以与 baseline 重复比较、后续证据能够推翻既有结论。

十一、局限性、稳健性与未决问题

  1. 外部证据可迁移性有限。 土耳其额度 RCT、美国信用卡 RD、墨西哥利率实验的市场、产品、剂量和窗口均不同，不能直接标定任何新的应用场景。
  2. 公开生产案例不足。 因果 ML/OPE/bandit 的一手生产成效远少于方法论文和供应商材料，不能把推荐/广告 benchmark 换算为信贷利润。
  3. 联合二维动作证据稀缺。 单独研究额度或价格较多，真正对额度 × 价格长期联合随机化的公开数据少。
  4. long-horizon reward 包含规范性选择。 利息、损失、资本、留存、投诉和客户福祉如何折现，不是算法从历史数据中可以自行决定的。
  5. 观测因果依赖不能仅由数据验证的假设。 DML/DR 不能修复未观测混杂或缺失支持；大样本和复杂模型不改变这一点。
  6. 干扰和多贷方竞争可能破坏标准假设。 竞品、资金预算和多平台借贷会让一个账户的动作与其他结果相互影响。
  7. 法规状态和适用范围会变化。 任何具体应用都必须按主体、产品和客户所在地重新确定约束，本文不提供法律结论。
  8. 本文刻意保持理论与通用设计层级。 因而不对任何特定产品给出财务承诺、动作步长、流量比例或项目排期。

理论设计仍需明确十个问题：目标是单期贡献还是长期 NPV；哪些约束必须逐笔硬执行；状态是否近似充分；是否存在真实随机概率；结果成熟窗口如何定义；后续动作如何污染结果；joint action 空间如何获得支持；未接受 offer 的结果如何定义；公平与客户待遇采用什么形式化约束；异常时如何证明 policy 已冻结并正确回退。

十二、最终结论：理论能力必须按“可审计 → 可识别 → 可评估 → 可学习”递进

12.1 理论设计的优先序

  • 先定义状态、动作、potential outcomes、时间零点和长期价值，不从算法选型开始；
  • 将合法性、合同、偿付能力和不可接受风险建模为硬 action set；
  • 为可观测状态与潜在状态分别定义表示，显式保留不确定性；
  • 用少量 joint action 建立可识别的主效应与交互效应，再讨论连续最优；
  • 将预测、因果效应、policy value 和端到端安全分开验证；
  • 让 baseline、不可识别和回退成为一等输出，而不是异常分支。

12.2 因果策略主张成立的通用条件

  • 每个决策事件在语义上包含当时状态、candidate action、分配、executed action 和原因；
  • 实验或 logging policy 具有 logged propensity，后续覆盖不改写原始值；
  • 主 estimand、动作对照、结果窗口和污染规则预先定义；
  • CATE/OPE 同时报告不确定性、overlap、ESS 和分群风险；
  • 候选 policy 在不支持区域明确回退 baseline；
  • 任一硬约束或关键证据缺失都使动作降级为不可执行/不可识别，而非由优化器补偿。

12.3 bandit 近似成立的必要条件

bandit 只有在动作对未来状态的影响可以忽略，或已经被可靠封装进单次 reward 时，才是正确的问题类型。即使如此，安全 action set、真实 propensity、延迟结果、稳定 baseline、可重放 policy 和独立约束仍只是必要而非充分条件。若动作会系统性改变下一期余额、风险、需求或可选动作，理论对象从一开始就是动态 regime/MDP/POMDP；此时应证明序列支持、转移与长期价值，而不是把更多远期标签塞进单步 reward。

最终判断不是“要不要用最先进算法”，而是：理论定义能否让每一次行动都可约束、可识别、可验证，并在证据不足时返回 baseline 或未知。 这才是额度—定价联动系统从评分卡时代迈向因果决策时代的分水岭。
